# Factum - Task Manager

Платформа для управления идеями и проектами. Позволяет пользователям создавать идеи, находить участников с нужными навыками, формировать команды и совместно работать над проектами.

**Статус:** В активной разработке

## О проекте

Factum решает проблему поиска единомышленников для реализации идей. Пользователи могут:
- Создавать идеи с описанием необходимых ролей (разработчик, дизайнер, маркетолог и т.д.)
- Искать участников по навыкам и интересам
- Откликаться на роли в чужих идеях
- Формировать команды для совместной работы

В будущем платформа расширится до полноценного инструмента управления проектами с системой задач, лентой событий, комментариями и чатом.

## Технологический стек

- **Backend:** Python 3.14, Django 5.x
- **База данных:** PostgreSQL
- **Тестирование:** pytest, pytest-django, pytest-cov, factory-boy
- **Архитектура:** Сервисный слой, разделение бизнес-логики и представления

## Текущий функционал

### Реализовано
- Регистрация и авторизация пользователей
- Профили с навыками (многие-ко-многим через промежуточную таблицу)
- Создание идей с описанием и ролями
- Поиск идей с фильтрацией
- Отклики на роли в идеях
- Система статусов для откликов (ожидает, принят, отклонен)
- Валидация бизнес-логики на уровне моделей
- Оптимизация запросов к БД (select_related, prefetch_related)
- Логирование ключевых событий

### В разработке
- Управление проектами на основе идей
- Система задач с назначением и отслеживанием
- Лента событий (activity feed)
- Комментарии к идеям и комментариям (вложенные)
- Система уведомлений
- Чат между участниками проектов

## Установка и запуск

### Требования
- Python 3.14+
- PostgreSQL 14+
- Git

### Локальная установка

1. Клонируйте репозиторий:
```bash
git clone https://github.com/danilov-dev/factum-task-manager
cd factum-task-manager
```

2. Создайте виртуальное окружение:
```bash
python -m venv venv
source venv/bin/activate  # Linux/macOS
# или
venv\Scripts\activate     # Windows
```

3. Установите зависимости:
```bash
pip install -r requirements.txt
```

4. Настройте базу данных:
```bash
# Создайте базу данных в PostgreSQL
createdb factum_db

# Примените миграции
python manage.py migrate
```

5. Запустите сервер разработки:
```bash
python manage.py runserver
```

Приложение будет доступно по адресу http://127.0.0.1:8000/

Проект покрыт тестами с использованием pytest.

### Запуск всех тестов:
```bash
pytest
```

### Запуск с отчетом о покрытии:
```bash
pytest --cov=apps --cov-report=term-missing
```

### Запуск конкретных тестов:
```bash

pytest tests/integration/   # Только integration-тесты

```

## Архитектура

Проект следует принципам чистой архитектуры:

```
apps/
├── ideas/
│   ├── models.py          # Модели данных
│   ├── services/          # Бизнес-логика
│   ├── views/             # Обработка HTTP-запросов
│   ├── forms/             # Формы валидации
│   └── urls.py            # Маршрутизация
└── users/
    ├── models.py
    ├── services/
    └── views/
```

### Ключевые принципы
- **Сервисный слой:** Бизнес-логика изолирована в `services/`, что упрощает тестирование и переиспользование
- **Тонкие контроллеры:** Views отвечают только за обработку HTTP, делегируя логику сервисам
- **Валидация на уровне моделей:** Бизнес-правила проверяются в методах `clean()` моделей
- **Оптимизация запросов:** Активное использование `select_related` и `prefetch_related` для избежания N+1 проблемы

## Структура базы данных

### Основные сущности
- **User** - пользователь с навыками (M2M через UserSkill)
- **Idea** - идея с автором и описанием
- **Role** - роль в идее (например, "Backend Developer")
- **IdeaResponse** - отклик пользователя на роль
- **Skill** - навык пользователя

### Связи
- User <-> Skill (многие-ко-многим)
- Idea -> User (автор)
- Idea -> Role (один-ко-многим)
- Role -> IdeaResponse (один-ко-многим)
